# 课程重编排建议

## 目标

- 不再把课程数量固定死在 8 节。
- 让知识依赖顺序更自然，减少“先讲结果、后补前提”的情况。
- 把原第 7 课拆开，分别讲清 `SFT` 和 `RL/偏好优化`。
- 保持“扫盲课”定位，但给后续扩展留接口。

## 当前结构的主要问题

- `Tokenizer / Embedding / Position Encoding / Attention` 这条主线被拆散了，导致学生不容易形成统一的“表示 -> 计算 -> 生成”认知框架。
- `Prompt` 在课程里出现得偏早，但“为什么 Prompt 有用”的前置解释还不够完整。
- `幻觉` 讲了问题，但没有把 `RAG / 工具 / 外部知识` 纳入一个完整解决框架。
- 原第 7 课信息密度过大，`SFT / RLHF / PPO / DPO / GRPO / GSPO` 混在一起，不适合扫盲课节奏。
- `多模态` 放在最后是合理的，但它需要前面更明确地铺垫“表示空间”和“对齐”的概念。

## 推荐目录（10 节）

### 第1课：概率与语言模型

- 核心问题：AI 为什么“会说话”
- 关键知识点：
  - Next-token prediction
  - Softmax
  - 交叉熵
  - 温度参数
- 目标：
  - 建立“AI 本质上是在做概率预测”的底层认知

### 第2课：Tokenizer 与向量表示

- 核心问题：文本是怎么变成模型能处理的输入的
- 关键知识点：
  - Tokenizer / BPE / 子词
  - Embedding
  - 词向量与语义空间
  - Position Encoding 的作用
- 目标：
  - 把“离散文本 -> 连续表示”的链条讲完整

### 第3课：注意力与 Transformer

- 核心问题：模型如何利用上下文
- 关键知识点：
  - Query / Key / Value
  - Self-Attention
  - Multi-Head Attention
  - Transformer 基本结构
  - KV Cache 的推理直觉
- 目标：
  - 让学生知道模型不是“背答案”，而是在上下文上做加权计算

### 第4课：预训练、规模化与能力提升

- 核心问题：模型为什么越训越强、越大越聪明
- 关键知识点：
  - Pre-training
  - Scaling Law
  - Chinchilla
  - In-context Learning
  - Emergence 的现象与争议
- 目标：
  - 解释能力从哪里来
  - 顺便把“Prompt 为什么会有效”铺垫出来

### 第5课：提示词工程与上下文学习

- 核心问题：为什么给不同提示词，模型表现差异这么大
- 关键知识点：
  - Role / Context / Task / Format / Constraints
  - Few-shot
  - Chain of Thought
  - Prompt 的边界
- 目标：
  - 把“使用模型”这一层讲实用
  - 同时避免把 Prompt 神化

### 第6课：幻觉、检索与工具使用

- 核心问题：AI 为什么会错，怎样让它更可靠
- 关键知识点：
  - 幻觉类型
  - 知识截止与训练数据局限
  - RAG 基本思路
  - Tool Use / Function Calling 的直觉
  - 人工校验与评测
- 目标：
  - 不只讲“问题是什么”，还讲“工程上怎么缓解”

### 第7课：指令微调（SFT）与模型“变听话”

- 核心问题：模型为什么从“会续写”变成“会按要求完成任务”
- 关键知识点：
  - Pre-training vs SFT
  - Instruction Tuning
  - 指令数据格式
  - IFT / SFT / LoRA
  - 多模态模型里的 SFT
- 目标：
  - 单独讲清“会做任务”这一步是怎么来的

### 第8课：偏好学习与强化学习后训练

- 核心问题：模型为什么不仅“能做事”，还会更“像人类偏好”
- 关键知识点：
  - Preference Data
  - Reward Model
  - RLHF
  - PPO 的直觉
  - DPO / GRPO / GSPO 的位置关系
  - Constitutional AI / RLAIF
- 目标：
  - 把“听指令”和“符合偏好/安全要求”区别开

### 第9课：多模态理解基础

- 核心问题：模型为什么能看图、读文档、理解多种输入
- 关键知识点：
  - 模态与表示空间
  - CLIP / 对比学习
  - 视觉编码器 + Adapter + LLM
  - VQA / 检索 / OCR / 文档理解
  - 多模态局限
- 目标：
  - 作为前 8 课的自然扩展，而不是孤立的新主题

### 第10课：图像生成基础（Diffusion / Flow Matching）

- 核心问题：模型为什么能“画图”，而且越画越像
- 关键知识点：
  - 图像生成和图文理解不是同一类任务
  - Diffusion 的基本直觉：逐步去噪
  - 条件生成：文本如何控制图像
  - Latent Diffusion 的工程意义
  - Flow Matching 的直觉：学一条从噪声到数据的连续路径
  - Diffusion vs Flow Matching：训练目标和采样思路的差异
- 目标：
  - 单独建立“生成式视觉模型”的认知，不和 VQA / OCR / 图文对齐混在一起

#### 建议讲法

- 先讲“为什么图像生成不能直接套用 next-token prediction 的直觉”
- 再讲 `Diffusion = 逐步加噪 / 去噪`
- 然后讲 `Latent Diffusion` 为什么更省算力
- 最后再讲 `Flow Matching`，作为“不是只有扩散一条路”的视角升级

#### 可纳入的代表工作

- `DiT`（Scalable Diffusion Models with Transformers, ICCV 2023, William Peebles, Saining Xie）
  - 适合用来讲：为什么 Transformer 也能做图像生成
  - 关键词：`Diffusion Transformer`
- `SiT`（Exploring Flow and Diffusion-based Generative Models with Scalable Interpolant Transformers, ECCV 2024, Nanye Ma et al., Saining Xie）
  - 适合用来讲：扩散模型和 flow / interpolant 视角之间的联系
  - 关键词：`flow vs diffusion`
- `Inference-Time Scaling for Diffusion Models beyond Scaling Denoising Steps`（CVPR 2025, Nanye Ma et al., Saining Xie）
  - 适合用来讲：图像生成模型也有“推理时扩展”问题，不只是 LLM 有 inference-time scaling
  - 关键词：`inference-time scaling`
- `Representation Alignment for Generation: Training Diffusion Transformers Is Easier Than You Think`（ICLR 2025）
  - 适合用来讲：训练 recipe 和表示对齐对生成质量的影响
  - 关键词：`representation alignment`
- `Diffusion Transformers with Representation Autoencoders`（ICLR 2026, Boyang Zheng et al., Saining Xie）
  - 适合用来讲：latent 表示质量会直接影响图像生成效果
  - 关键词：`RAE / latent representation`

#### 课堂建议

- 扫盲主线里不建议把这几篇论文都讲成“论文精读”
- 更好的做法是拿其中 2-3 篇做“技术演进案例”
- 推荐主线组合：
  - `DiT`：建立 Transformer 做图像生成的直觉
  - `SiT`：引出 flow matching / interpolant 视角
  - `Inference-Time Scaling` 或 `RAE`：说明这条路线近两年还在快速演进

## 为什么建议单独加一节图像生成

原因很简单：`多模态理解` 和 `图像生成` 虽然都处理图像，但它们解决的问题并不一样。

- 多模态理解：重点是“看懂”
- 图像生成：重点是“生成”
- 前者更多连接到 `CLIP / 表示对齐 / 视觉编码器`
- 后者更多连接到 `Diffusion / Latent Space / 采样过程`

如果把这两类内容混在同一节，学生很容易产生两个误解：

- 误以为“会看图”和“会画图”是同一种模型能力
- 误以为图像生成也主要靠 Transformer/LLM 那套机制

所以如果课时允许，我更推荐拆开。

4. 预训练、规模化与能力提升

7. 偏好学习与 RL 后训练

## 如果想压回 8 节

最合理的压缩方式不是把第 7、8 课再合回去，而是把第 5、6 课合并。

### 8 节压缩版

1. 概率与语言模型
2. Tokenizer、Embedding 与位置编码
3. 注意力与 Transformer
4. 预训练、规模化与能力提升
5. Prompt、幻觉与工具使用
6. SFT：从续写到按指令完成任务
7. 偏好学习与 RL 后训练
 8. 多模态与图像生成导论

原因：

- `Prompt` 和 `幻觉` 都属于“用户如何和模型协作”的应用层。
- `SFT` 和 `RL` 最好分开，否则学生会误以为“对齐 = 强化学习”。
- `多模态` 仍然适合放在最后，作为前面所有概念的综合应用。
- 如果只有 8 节，`Diffusion / Flow Matching` 只能讲直觉，不能讲太深。

## 每节课的认知主线

- 第1-3课：模型怎么接收输入、表示信息、利用上下文
- 第4课：能力从哪里来
- 第5-6课：人怎么更好地使用模型、约束模型
- 第7-8课：模型是如何被训练成“更会做事、更符合偏好”的
- 第9课：模型如何理解图像与文档
- 第10课：模型如何生成图像

## 改造顺序建议

1. 先改课程目录和每课“上节回顾 / 下节预告”。
2. 把原第 3 课扩成 `Tokenizer + Embedding + Position Encoding`。
3. 把原第 5 课扩成 `幻觉 + 检索 + 工具使用`。
4. 把原第 7 课拆成两课：
   - 第7课：SFT / IFT / Instruction Tuning
   - 第8课：Preference Learning / RLHF / DPO / GRPO / GSPO
5. 把原第 8 课重命名为“多模态理解基础”。
6. 新增“图像生成基础：Diffusion / Flow Matching”。

## 结论

推荐直接从“8课深化版”升级为“10课版”。

核心原因不是为了多一课，而是为了把这两个不同问题拆开：

- `SFT` 解决“模型会不会按任务要求输出”
- `RL / Preference Optimization` 解决“模型更偏好什么样的输出”
- `Multimodal Understanding` 解决“模型能不能看懂图像”
- `Diffusion / Flow Matching` 解决“模型能不能生成图像”

这些问题混讲，学生通常会混淆。
