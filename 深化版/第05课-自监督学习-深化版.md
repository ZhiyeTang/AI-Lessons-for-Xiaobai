# AI 扫盲课 · 深化版

## 第5课：自己教自己——自监督学习

---

## 复习回顾

### 前四课我们学了AI的结构

1. **概率建模** → 按概率选词
2. **注意力机制** → 回头看
3. **向量嵌入** → 词变坐标
4. **分层学习** → 逐层抽象

**新问题**：
AI是怎么学会这么多知识的？谁教它的？

---

## 传统机器学习的痛点

### 以前教AI识别猫

1. 找100万张猫的照片
2. 找人**手工标注**："这是猫"
3. 把照片和标签一起喂给AI

**问题**：
- 标注100万张？太贵了！💰
- 很多任务没法标注（比如"理解情感"）

---

## 自监督学习的聪明之处

### 不需要人工标注！

**让数据自己产生标签**

🎮 **玩"填空游戏"**：

> "今天天气很好，我去公园散步，看到了很多___。"

遮住一个词让AI猜：
- 猜对了 → 说明理解上下文
- 猜错了 → 调整，下次争取猜对

**重复万亿次...**

---

## AI从填空游戏中学到了什么？

### 表层

- **语法**："天气很"后面跟形容词
- **搭配**："天气好"比"天气桌子"合理

### 深层

- **世界知识**："公园"和"花"经常一起出现
- **逻辑**："因为天气好，所以去公园"
- **推理**：能猜出被遮住的词

---

## 为什么叫"预训练"？

### 第一步：预训练

让AI先"读遍互联网"，学会语言规律

**GPT-3的训练数据**：
- 约5000亿个token
- 所有维基百科（几百万遍）
- 大量书籍、网页、代码

### 第二步：微调（Fine-tuning）

用少量标注数据，学会具体任务

---

## 【新增】MLLM预训练的详细过程

### 来自Notion MLLM知识库

MLLM（多模态大语言模型）的预训练通常采用**两阶段策略**：

```
阶段1：预训练（Pre-training）
┌─────────────────────────────────────────────────┐
│ 目标：训练Adapter，让视觉特征对齐到文本空间        │
│ 数据：图像-文本对（Image-Caption Pairs）          │
│ 训练对象：主要训练Adapter，LLM和Encoder通常冻结   │
└─────────────────────────────────────────────────┘
                      ↓
阶段2：指令微调（Instruction Fine-tuning）
┌─────────────────────────────────────────────────┐
│ 目标：让模型能够完成用户的不同任务                │
│ 数据：指令-输入-输出三元组                        │
│ 训练对象：同时微调Adapter和LLM                   │
└─────────────────────────────────────────────────┘
```

---

## 【新增】预训练阶段详解

### 来自Notion MLLM知识库

**预训练的核心目标**：训练Adapter（模态对齐投影器）

**训练数据**：
- 图像-文本配对数据（如LAION-5B、COYO-700M）
- 格式：`[图像, "一只猫坐在沙发上"]`

**训练过程**：
1. Encoder提取图像特征：$I_f = \text{Encoder}(I)$
2. Adapter将视觉特征投影到LLM空间：$V = \text{Adapter}(I_f)$
3. LLM根据视觉特征生成文本描述
4. 最小化生成文本与真实描述的差异

**大白话理解**：
就像教一个翻译官，看着图片能说出这是什么。先让Adapter学会"看图说话"的基本能力。

---

## 【新增】MLLM预训练 vs LLM预训练

### 来自Notion MLLM知识库

| 对比项 | LLM预训练 | MLLM预训练 |
|-------|----------|-----------|
| **输入数据** | 纯文本 | 图像-文本对 |
| **训练对象** | 整个模型 | 主要是Adapter |
| **预训练目标** | 预测下一个token | 对齐视觉-文本空间 |
| **冻结参数** | 无 | Encoder和LLM通常冻结 |
| **可训练参数** | 全部 | Adapter + 可选LLM LoRA |
| **数据规模** | 数万亿token | 数亿-数十亿图像-文本对 |

**关键区别**：
- LLM是从头学习语言
- MLLM是让已有LLM"学会看"

# 数学原理：掩码语言建模

---

## Masked Language Model (MLM)

### BERT的训练目标

**输入**：
```
今天天气[MASK]，我去公园散步。
```

**目标**：预测被掩盖的词

$$\mathcal{L}_{\text{MLM}} = -\mathbb{E}_{x \sim \mathcal{D}} \log P(x_{\text{mask}} | x_{\backslash \text{mask}})$$

**掩码策略**：
- 80% 概率替换为 [MASK]
- 10% 概率替换为随机词
- 10% 概率保持不变

---

## 自回归语言建模 (GPT)

### 预测下一个词

$$\mathcal{L}_{\text{LM}} = -\sum_{t=1}^{T} \log P(x_t | x_{1:t-1})$$

**因果掩码（Causal Masking）**：
- 每个词只能看到前面的词
- 三角形注意力掩码

```
      t1 t2 t3 t4
t1    [1  0  0  0]
t2    [1  1  0  0]
t3    [1  1  1  0]
t4    [1  1  1  1]
```

**适合生成任务**，从左到右逐个生成

---

## 【新增】CLIP对比学习的深入解释

### 来自Notion MLLM知识库

CLIP（Contrastive Language-Image Pre-training）是多模态预训练的核心方法。

**核心思想**：
让配对的图像-文本在特征空间中靠近，不配对的远离。

**数学公式**：

$$\mathcal{L}_{\text{CLIP}} = \frac{1}{2}(\mathcal{L}_{I} + \mathcal{L}_{T})$$

其中图像到文本的损失：
$$\mathcal{L}_{I} = -\frac{1}{N}\sum_{i=1}^{N} \log \frac{\exp(\text{sim}(I_i, T_i)/\tau)}{\sum_{j=1}^{N} \exp(\text{sim}(I_i, T_j)/\tau)}$$

---

## 【新增】CLIP损失的详细推导

### 来自Notion MLLM知识库

**完整计算流程**：

```python
# 1. 分别提取图像特征和文本特征
I_f = image_encoder(I)      # [n, d_i]
T_f = text_encoder(T)       # [n, d_t]

# 2. 线性投射到相同维度 + L2归一化
I_e = l2_normalize(I_f @ W_i, axis=1)  # [n, d]
T_e = l2_normalize(T_f @ W_t, axis=1)  # [n, d]

# 3. 计算缩放的余弦相似度 [n, n]
logits = I_e @ T_e.T * exp(t)  # t: 温度参数

# 4. 对称的对比学习损失（InfoNCE）
labels = arange(n)  # 对角线是正样本
loss_i = cross_entropy_loss(logits, labels, axis=0)  # I2T
loss_t = cross_entropy_loss(logits, labels, axis=1)  # T2I
loss = (loss_i + loss_t) / 2
```

---

## 【新增】CLIP的InfoNCE损失

### 来自Notion MLLM知识库

**InfoNCE损失的本质**：
把$N$个图像-文本对看作$N$个类别的分类问题。

对于第$i$个图像：
- 正样本：配对的第$i$个文本
- 负样本：其他$N-1$个文本

$$\mathcal{L}_{I2T}^{(i)} = -\log \frac{\exp(\text{sim}(I_i, T_i)/\tau)}{\underbrace{\exp(\text{sim}(I_i, T_i)/\tau)}_{\text{正样本}} + \underbrace{\sum_{j \neq i} \exp(\text{sim}(I_i, T_j)/\tau)}_{\text{负样本}}}$$

**温度参数$\tau$的作用**：
- $\tau$小 → 分布更尖锐，强调最难的负样本
- $\tau$大 → 分布更平滑，关注所有负样本

---

## 【新增】对比学习的直观理解

### 来自Notion MLLM知识库

**想象一个图书馆**：

```
       文本空间
         🐱 "一只猫"
        /
图像空间 🐱 → 应该靠近
        \
         🐶 "一只狗" → 应该远离
```

**目标**：
- 配对的图像和文本 → 向量距离近（余弦相似度≈1）
- 不配对的图像和文本 → 向量距离远（余弦相似度≈0或负）

**为什么叫"对比"**：
模型通过对比正负样本的相对距离来学习表示。

---

## 数据量与模型性能

### Scaling Law的启示

**Kaplan et al., 2020 (OpenAI)**

$$L(N) \propto N^{-0.076}$$

损失随模型参数 $N$ 增大而下降

**训练数据量**：
- GPT-3: 3000亿 token
- PaLM: 7800亿 token
- LLaMA-2: 2万亿 token

**核心发现**：
- 更多数据 + 更大模型 = 更好性能
- 这种关系是可预测的

---

## 预训练 vs 微调

### 两阶段学习

**预训练（Pre-training）**：
- 大规模无标注数据
- 学习通用语言表示
- 计算成本高（数百万美元）

**微调（Fine-tuning）**：
- 小规模标注数据
- 适应特定任务
- 计算成本低（数千美元）

**迁移**：预训练学到的知识迁移到下游任务

# 经典论文：BERT与预训练革命

---

## BERT (2018)

### Devlin et al., Google

**论文**："BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding"
NAACL 2019 Best Paper

**核心创新**：

1. **双向编码器**
   - 每个词可以看到左右上下文
   - vs GPT只向左看

2. **Masked Language Model**
   - 填空任务训练
   - 被迫理解上下文

3. **Next Sentence Prediction**
   - 判断句子B是否接在句子A后
   - 学习句子级关系

---

## BERT vs GPT vs ELMo

| 特性 | ELMo | GPT | BERT |
|------|------|-----|------|
| 架构 | Bi-LSTM | Transformer Decoder | Transformer Encoder |
| 方向 | 双向（独立） | 单向（左→右） | 双向（真双向） |
| 预训练任务 | LM | LM | MLM + NSP |
| 词向量 | 上下文相关 | 上下文相关 | 上下文相关 |
| 年份 | 2018 | 2018 | 2018 |

**BERT的关键优势**：
真正双向，融合左右上下文

---

## GPT系列演进

### 从GPT-1到GPT-4

| 模型 | 年份 | 参数量 | 关键特性 |
|------|------|--------|----------|
| GPT-1 | 2018 | 1.17亿 | 生成预训练 + 微调 |
| GPT-2 | 2019 | 15亿 | 零样本能力 |
| GPT-3 | 2020 | 1750亿 | 上下文学习 |
| GPT-4 | 2023 | 未公开 | 多模态 |

**GPT-3的惊人发现**：
- 不需要微调
- 给几个示例（few-shot），就能学会新任务
- 这是"涌现"能力的体现
