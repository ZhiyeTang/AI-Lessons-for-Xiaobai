## 什么是多模态？

### 模态（Modality）

信息的**表现形式**：
- **文本**：文字、语言
- **图像**：照片、图画
- **音频**：声音、音乐
- **视频**：图像+音频+时间

### 多模态 = 同时处理多种信息

**例子**：
- 你看电影 = 视觉 + 听觉
- 你读绘本 = 图像 + 文字

**AI的目标**：像人一样，同时"看懂"和"读懂"

---

## 为什么多模态很难？

### 问题1：不同模态的"语言"不同

| 模态 | 表示方式 | 处理难度 |
|------|----------|----------|
| 文本 | 离散的token序列 | ⭐⭐ |
| 图像 | 连续的像素矩阵 | ⭐⭐⭐ |
| 音频 | 时频谱图 | ⭐⭐⭐ |
| 视频 | 图像序列+音频 | ⭐⭐⭐⭐ |

### 问题2：如何"对齐"不同模态

- "一只猫"（文本）↔ 🐱（图像）
- 如何让AI知道它们是同一个东西？

### 问题3：计算量巨大

- 一张图 = 100万像素 × 3通道 = 300万数字
- 一段文字 = 1000个token
- 两者一起处理 = 算力爆炸 💥

---

## 解决方案：CLIP模型

### CLIP：Connecting text and images

**OpenAI, 2021**

**核心思想**：
在**同一个向量空间**中表示图像和文本

```
图像 → 图像编码器 → 图像向量 ──┐
                                   ├── 对比学习 → 相似度
文本 → 文本编码器 → 文本向量 ──┘
```

### 训练方法：对比学习

1. 拿一张图和对应的文字描述
2. 让它们的向量**靠近**
3. 拿不匹配的图和文字
4. 让它们的向量**远离**

**例子**：
```
🐱 + "一只猫" → 向量距离近 ✅
🐱 + "一辆汽车" → 向量距离远 ❌
```

## CLIP的革命性意义

### 1. 零样本分类

不需要针对特定任务训练！

```
输入图片
↓
生成描述候选：
- "这是一只猫"
- "这是一只狗"
- "这是一辆汽车"
↓
选相似度最高的
```

### 2. 跨模态检索

- **以文搜图**：输入"夕阳下的海滩"，找相关图片
- **以图搜文**：输入图片，生成描述

### 3. 多模态理解的基础

现代多模态大模型（GPT-4V、Qwen-VL）都基于类似思想

## 从CLIP到多模态大模型

### 典型架构

```
图像 → 图像编码器（CLIP视觉部分）
       ↓
    Adapter（适配器）
       ↓
文本 → 文本编码器（LLM）
       ↓
    融合 → 生成回答
```

**关键组件**：
1. **视觉编码器**：把图像变成向量
2. **适配器**：对齐视觉和文本的维度
3. **大语言模型**：理解和生成文本

## 多模态的局限

### 局限1：视觉幻觉

- AI可能"看错了"
- 把熊猫看成狗
- 数错图中物体数量

### 局限2：空间理解弱

- "左边"、"上方"等方位词理解不准
- 复杂场景容易混乱

### 局限3：细节丢失

- 图像编码会压缩信息
- 小文字、细微特征可能丢失

### 局限4：推理能力有限

- 能"描述"，但难"推理"
- 比如："如果图中的人往前走，会发生什么？"
